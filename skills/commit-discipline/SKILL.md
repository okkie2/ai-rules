# Commit Discipline

## Rules

Use conventional commit prefixes such as `feat:`, `fix:`, `refactor:`, `docs:`, and `chore:`.

Make each commit represent one logical change and remain understandable in isolation.

Avoid mixed concerns in one commit.

Avoid vague messages such as `update` or `stuff`.

## Good commit example

`feat: add local transcription benchmarking runner`

## Bad commit example

`update files`
