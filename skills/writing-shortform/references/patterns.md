# Shortform Pattern Library

Use these as selectable shapes, not as rigid formulas.

## Belief -> Breakdown -> Rule

Core move:
Take something people assume is true, break it, and replace it with a rule.

Useful when:
- the audience is relying on a bad default
- the post needs a strong contrarian opening

## Surface -> Hidden Layer -> Consequence

Core move:
Expose what sits underneath something that looks fine.

Useful when:
- the visible story is misleading
- stress conditions reveal the real system behaviour

## Frame -> Reframe -> Compression

Core move:
Change how the problem is framed, then compress it into a simple heuristic.

Useful when:
- debate is stuck in the wrong binary
- a cleaner decision rule is needed

## Goal -> Anti-goal -> Mirror

Core move:
Define failure clearly, then show that current behaviour already produces it.

Useful when:
- institutions or teams are violating their own stated aim
- the post should expose contradiction rather than argue abstractly

## Continuum -> Threshold -> Regime Change

Core move:
Show that something is not linear and flips after a threshold.

Useful when:
- scale, complexity, or dependence changes behaviour qualitatively

## Asset -> Dependency -> Control Test

Core move:
Challenge ownership or capability by exposing underlying reliance.

Useful when:
- control, sovereignty, lock-in, or resilience is central

## Signal -> Metric -> Behaviour

Core move:
Show how measurement drives behaviour that diverges from the desired outcome.

Useful when:
- incentives, KPIs, dashboards, or compliance measures are distorting reality
