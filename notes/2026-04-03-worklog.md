# Worklog - 2026-04-03

## Purpose

This note is a human-readable worklog for today's work in `ai-rules`.
It complements the short factual entries in `CHANGELOG.md`.

## Summary

Today we turned `ai-rules` into a real source-of-truth repository for shared AI-agent guidance instead of leaving that guidance only in the live Codex runtime.

We brought the existing curated shared skills and rules into the repository, connected the repository to GitHub, and established a safer operating model in which the repository is canonical and the live runtime under `~/.codex` is a deployment target.

We then added a deploy path for that model. This included sync scripts for repo-to-runtime and limited runtime-to-repo recovery, plus a dedicated `ai-rules-sync` workflow skill so the sync process itself is governed like the rest of the system.

We also made the repository much easier to use and maintain. `MANUAL.md` now explains setup, syncing, and the current curated shared skills. `README.md` was tightened into a shorter entrypoint. Later, standing rules were separated from open work by introducing `DOALWAYS.md`, so `TODO.md` can stay focused on unfinished actions.

On the skill side, the biggest functional change was adding `application-development` as the first generic coding context skill. That matters because coding is now represented not only by review, routing, and hygiene skills, but also by an actual implementation context in the shared skill system.

Finally, we tightened governance around the repository itself. We clarified that `TODO` and standing rules are different things, put the broader `TODO` versus `DOALWAYS` pattern into backlog and roadmap, and left the repository in a cleaner state for future work such as Claude support, deeper coding contexts, and skill evals.

## Main Outcomes

- `ai-rules` now acts as the versioned source of truth for curated shared Codex skills and governance notes.
- Sync between the repository and `~/.codex` is now scripted and documented.
- The repository now has a practical manual and a clearer documentation split.
- `application-development` was added as the first generic coding context skill.
- Standing rules were separated from open tasks by introducing `DOALWAYS.md`.
